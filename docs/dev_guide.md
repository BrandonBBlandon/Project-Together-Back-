la guia de desarrollo para el aplicativo:



se manejan dos repositorios, uno para el front y otro para el back,
ramas para cada repositorio utilizables: dev, testing, main
la base de datos esta en el motor de base de datos pero los cambios reflejan tareas para el back y posiblemente para front
flujo basico
si existe un problema: dejarlo como una issue en github
si falta algo pendiente tambien dejarlo como una issue pendiente

si se quiere desarrollar se realiza un git pull en la rama dev  algo o no se logro terminar algo se puede comentar en la issue hasta donde se pudo resolver
al terminar de desarrollar subir a la rama de testing y dejar issue de pruebas completas
al terminar pruebas completas subir a la rama main que va a estar conectada al deploy oficial
